# Clinical AI Extraction Pipeline: Summary Report

## 📋 Project Overview
The **Clinical-AI Extraction Pipeline** is an automated system designed to solve a critical bottleneck in oncology: the manual extraction of patient data from **Multidisciplinary Team (MDT)** meetings. Currently, vital clinical information is "trapped" in semi-structured Word documents, making it difficult to audit or research patient outcomes.

**Objective:** Transform 50+ synthetic clinical MDT proformas into a structured, longitudinal Excel database that aligns with NHS Digital Technology Assessment Criteria (DTAC).

---

## 🏗 Repository Structure
- **`data/`**: Contains the source Word documents (`hackathon-mdt-outcome-proformas.docx`) and the "ground truth" Excel prototype (`hackathon-database-prototype.xlsx`).
- **`baseline-solution/`**: Contains the core logic and previous iterations of the extraction pipeline.
- **`src/`**: Python source code for parsing and normalization.
- **`docs/`**: Technical specifications, clinical context, and meeting minutes.
- **`tests/`**: Automated validation suites to ensure extraction accuracy.

### Key Achievements:
- **Hybrid-Lite Architecture (v4):** Implemented a multi-pass system that fuses structural table data with paragraph-level context (recovering previously "trapped" MDT dates).
- **Clinical Safety:** Integrated **MedSPaCy** for professional negation detection (e.g., distinguishing "M0" from "Metastases").
- **Zero-Hallucination Identifiers:** Forced string encoding in Excel to eliminate scientific notation and `.0` suffixes on NHS numbers and MRNs.
- **Improved Initials Logic:** Implemented layout-aware name parsing to correctly extract patient initials (e.g., Aiden O'Connor -> AO).

## 🔍 Identified Gaps & Challenges
Despite strong performance, the following areas require further development:
1.  **Treatment Specifics:** Extraction of complex Chemotherapy, Radiotherapy, and Immunotherapy cycles is currently limited.
2.  **Pathology Nuance:** capturing secondary MRI results and specific histology biopsy dates needs refinement.
3.  **Reliability at Scale:** Ensuring the heuristics remain robust across all 50+ diverse clinical cases.
